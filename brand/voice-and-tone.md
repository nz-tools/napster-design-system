# Voice and Tone

Napster writes like **a smart magazine about a serious idea** — editorial, declarative, unhedged. Never like a product marketer. The voice predates the design system; the design system serves the voice.

## Five load-bearing patterns

### 1. Declarative sentences. Short ones.

> *Software just became labor.*
> *A chatbot answers questions. A companion mentors.*

Hedging betrays the brand. Verbs like "we believe", "we think", "we feel" rarely belong in marketing copy. The product makes the claim; the copy states it.

### 2. Contrast pairs — the signature move.

Old state / new state, in parallel:

> *Before: Software = Tool you operate.*
> *Now: Software = Worker you manage.*

The before-line should be brief enough to land in one breath. The after-line should not be longer than the before-line.

### 3. Specific numbers, never vague claims.

> *$5/hour vs the industry's $1/minute.*
> *Your first crew member is live in under an hour.*
> *2 million sessions per month.*

Not: *"industry-leading"*, *"at scale"*, *"transformative"*. If you don't have the number, find one. If you can't find one, write a different sentence.

### 4. Rename the category. Don't compete inside it.

We aren't a better chatbot — we're a **companion**. A **crew**.

You **hire**, **deploy**, **calibrate**, **tune**, **dial in**, **bring online**, **ship**. Crew members **remember**, **build on**, **pick up where**. The vocabulary is colleague-vocabulary, not product-vocabulary.

### 5. Name the invisible.

Start copy by naming a tension the reader hasn't articulated. Often by naming a group nobody talks about.

> *Every campus has a visible minority and an invisible majority.*
> *Every team has a Slack channel for the founders and an inbox for everyone else.*

The reader recognizes the picture before they recognize the product.

## Voice-check

| Axis | Rule |
|---|---|
| **Casing** | Sentence case for headlines. UPPERCASE for eyebrows. Never title case, except for proposal titles and section chrome. |
| **Person** | **You** for the user (benefit-led). **We** rarely — Napster is the system, not the speaker. |
| **Tense** | Present tense. Active verbs. *"Companions remember."* not *"will be able to remember."* |
| **Periods** | Hero lines end with a period. *"A specialist for every moment."* Signature. |
| **Em-dashes** | Sparingly — one per passage max, used for pivots, not lists. |
| **Exclamation marks** | Almost never. The declarative carries its own emphasis. |
| **Emoji** | **Never.** Not in product, not in marketing, not in decks. |
| **Numbers + dates** | Specific. Year stamped in footers ("2026"). Page numbers zero-padded ("00"–"99") on slides. |

## Banned words

Do not ship copy containing any of the following:

`revolutionize` · `disrupt` · `game-changing` · `AI-powered` · `leverage` ·
`solution` · `seamlessly` · `effortlessly` · `powerful` · `unlock` (the verb) ·
`empower` · `industry-leading` · `world-class` · `chatbot` (that's what we aren't) ·
`assistant` · `bot` · `users` (name them: students, operators, counselors, fans).

If a sentence depends on one of these words, the sentence is broken. Rewrite the sentence.

## Preferred vocabulary

| Concept | Use |
|---|---|
| AI agent | **companion** |
| Group of agents | **crew** |
| Onboarding an agent | **hire, deploy, ship, bring online** |
| Tuning behavior | **calibrate, tune, dial in** |
| Persistent memory | **remembers, builds on, picks up where** |
| Knowledge | **expertise, tacit knowledge, the hidden curriculum** |
| Teaching relationship | **mentorship** |
| Person using the product | **student, operator, counselor, founder, fan** — by role, never "user" |
| Customer-facing operator (Station) | **operator** (airport-lounge operator, hotel operator) |

## Named characters

Companions get full human names. Always.

- **Kai Mercer** — Chief of Staff.
- **Kevin Jones** — Finance.
- **Jordan Carter** — Fitness.
- **May Li** — Creative.

We talk about them like colleagues. Never *"Kai the AI"*. Never *"your AI assistant Kai"*. Just *"Kai"*. Full roster: `../imagery/people/persona-roster.md`.

## Hero headline anatomy

A canonical Napster hero is three parts:

1. **Eyebrow** — UPPERCASE / IBM Plex Mono 500 / 12px / `+0.48px` tracking / `#DD52CB`. Single line. 1–3 words.
2. **Hero headline** — Inter 700 / 72px / sentence case / ends with a period.
3. **Subtitle** — Inter 400 / 18px / `rgba(255,255,255,0.5)` / one sentence.

Example:

```
MEET YOUR CREW
AI companions you can see, talk to, and create with.
A specialist for every moment, available in seconds.
```

Within the hero, the load-bearing noun gets an accent treatment. Two options, never both on the same word:

- **`.gradient-word`** — the white-to-pink gradient. The default treatment. *"…create with."* / *"…every moment."*
- **`.accent-serif`** — Instrument Serif italic. The editorial alternative. Use when the headline already carries its own visual weight, or when the accent word is a name or noun that wants a literary register. *"Meet [Kai Mercer]."* with the name in italic serif.

## Companion names in prose

Companions are introduced with full human names in Instrument Serif italic — *Kai Mercer*, *Kevin Jones*, *May Li* — at first mention in any editorial passage. Subsequent mentions use the first name only and stay in the surrounding type. Never set the role label in italic serif; the serif gear is reserved for the name itself.

## Metadata register

Page numbers (`02 / 14`), timestamps (`14:32 UTC`), footer markers (`DESIGN SYSTEM · 2026 · v1.1.0`), spec values inside dense charts — all sit in IBM Plex Mono. The mono signals "this is a fact, not a claim." Body prose stays in Inter; the mono is reserved for what the eye scans, not what the eye reads.

## When in doubt

- If the copy could appear on a competitor's site verbatim, rewrite it.
- If the copy would embarrass a smart editor, rewrite it.
- If the copy hedges, cut the hedge.
- If the copy is generic, name the specific.
